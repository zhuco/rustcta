# RustCTA 0.3.6 更新日志

发布日期：2026-05-30

## 核心目标

本版本围绕交易所网关统一、资金费率套利实盘准备、跨交易所套利执行稳定性和对冲网格持续运行做了一轮工业化整改。目标是在不破坏现有对冲网格、资金费率策略和跨所套利策略运行方式的前提下，把交易所能力逐步沉淀到统一网关层。

## 主要更新

1. 版本号从 `0.3.5` 升级到 `0.3.6`。
2. 新增资金费率套利策略运行入口与配置模板，支持启动扫描、候选通知、实盘执行前置校验和按交易所统计套利结果。
3. 完善交易所统一网关目录结构，补充 `gateway`、`registry`、交易所配置抽象和策略侧调用边界。
4. 丰富 Binance、Bitget、Gate、OKX 等交易所的永续合约私有交易能力接入，补齐下单、撤单、查询、持仓、余额、用户流和错误归一化相关逻辑。
5. 扩展 Bitget、Gate、HTX 等行情适配器和私有永续协议能力，为后续独立开源交易所网关打基础。
6. 加强跨交易所套利执行链路，补充手续费审计、订单管理、私有同步、风控、仓位、机会筛选、执行统计和监控面板字段。
7. 保持对冲网格策略原启动方式不变，并验证 `multi_hedged_grid` 可在服务器重启后继续运行。

## 运行影响

- 已运行进程不会自动加载新代码，只有重启对应 systemd 服务或进程后才会使用 `0.3.6` 二进制。
- 本版本已重启并验证 `rustcta-multi-hedged-grid.service`，服务状态为运行中。
- 资金费率策略和跨所套利策略的运行方式保持兼容，但后续重启后会进入新的统一网关能力边界。

- `cargo build --release --bin rustcta`
- 服务器上传并重启 `rustcta-multi-hedged-grid.service`
- 查看 systemd 状态和 `logs/runtime/multi_hedged_grid_usdc_systemd.log`，确认 3 个 USDC 交易对正常挂单
