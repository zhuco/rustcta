# 订单流做市策略整改计划（RustCTA）

## 背景
- 已有功能：固定梯度做市（含盘口冲击检查）、盘口不平衡/OFI 偏移、库存偏移、价跳/OFI 重挂、事件轮询唤醒、风险缩量/停机、基础 PnL 估计、批量补单。
- 问题：缺少实时事件驱动、精细风控、可观测性、鲁棒生命周期，以及动态定价/库存管理的更多输入。

## 目标
- 建立高性能、事件驱动的订单流做市引擎，具备完备风控、指标监控、恢复与回放能力，满足低延迟和稳健性要求。

## 里程碑与任务
### M1：数据与事件驱动
- 接入交易所用户流与订单簿增量 WebSocket，落地本地 orderbook/队列位次（含重连与快照校验）。
- 将成交/订单状态变更作为触发器，替代轮询补单；保留 REST 回退路径。
- 增加幂等 clientOrderId 生成与重复过滤，提供速率/权重预算器。

### M2：风控与仓位治理
- 补齐风险阈值：未实现亏损、日亏、连续亏损、资金费率/波动率/滑点、保证金检查、熔断。
- 引入仓位成本与双向模式精确计算，支持按持仓侧分摊费用/资金费率。
- 增加挂单 TTL/队列位次限制、库存 soft/hard cap、速率限制策略。

### M3：定价与执行优化
- 动态价差/尺寸：结合波动率、趋势、外盘/跨市场提示、OFI/队列位置进行偏移与缩放。
- 事件驱动撤换价与补单：盘口跳变/队列塌陷/成交后即时撤换价，避免长时间挂在劣位。
- 增加滑点/冲击限额与局部止损（单侧/单批）。

### M4：可观测性与运维
- 指标与日志：延迟、补单命中率、撤单失败率、滑点、队列位次、库存/PNL、风控触发次数；暴露 Prom/日志结构化输出。
- 告警：风险触发、连接异常、重连退避耗尽、残单清理失败。
- 状态快照：本地持久化挂单/持仓/成本，进程重启后快速恢复。

### M5：验证与回放
- 单元/集成测试覆盖：orderbook/OFI、风险分支、事件驱动补单、TTL/重挂。
- 录制/回放：深度+成交日志回放，验证延迟与收益；压力/延迟测试脚本。
- 灰度上线 checklist：小额度单标的、熔断开启、指标监控、回归测试完成。

## 交付清单
- 代码：事件驱动引擎、风控扩展、动态定价模块、监控指标与告警。
- 配置：风控阈值、TTL、队列/滑点限制、监控开关、重连策略。
- 文档：运行与灰度指引、监控项说明、回放/压测指南、已知限制。
