# RustCTA v0.3.3 更新说明

发布日期：2026-04-20

本次版本聚焦于交易系统能力扩展、策略体系补强、账户监控与运行稳定性改进，覆盖现货/合约交易所接入、对冲网格、趋势与做市相关模块，以及独立仓位报告能力。

## 重点更新

### 1. 独立账户仓位报告

- 新增独立可执行程序，用于周期性汇总账户总权益、多头/空头敞口、USDT 价值、分交易对持仓与未实现盈亏。
- 支持企业微信 Markdown 推送，默认按 `.env` 中的交易所密钥启动，不依赖策略进程。
- 增加中文报告文案，便于直接用于实盘值守与巡检。

### 2. 对冲网格能力增强

- 新增与补充多套 `solusdc_hedged_grid` 配置，覆盖 `SOL/USDC`、`BNB/USDC`、`ADA/USDC`、`ETH/USDC` 等交易对。
- 对冲滚动网格增加更细的库存快照、成交箱、批量挂单与重启日志，便于定位吃单、补单、单边累仓与重建异常。
- 增加固定价差与按交易对精度配置的支持，提升不同标的在 Binance USDC 永续上的可复用性。

### 3. 监控与运行辅助

- 新增监控服务与相关配置，用于统一观察策略、账户、持仓和运行状态。
- 补充多份策略配置与运维文档，覆盖网格、均值回归、订单流、做市、趋势等模块。
- 增强日志与运行时输出，提升线上排障效率。

### 4. 策略体系扩展与重构

- 补充/重构 `hedged_grid`、`grid_scale`、`market_making`、`orderflow`、`sideways_martingale`、`solusdc_hedged_grid` 等策略模块。
- 趋势相关模块持续拆分，细化为进场、执行、市场数据、订单追踪、用户流等组件，便于后续演进。
- 均值回归、泊松做市、自动剥头皮等策略模块同步更新风险、执行与状态管理逻辑。

### 5. 交易所与基础设施改进

- Binance、Bitmart、Hyperliquid、OKX 适配层有持续更新，提升接口兼容性与账户/订单处理能力。
- 核心类型、账户管理器、风控、时间同步、Webhook、指标与统一日志模块继续完善。
- 新增 Mock 交易所支持，为离线调试与集成测试提供基础。

## 文档更新

本次补充或更新了多份中文文档，包括但不限于：

- 双向对冲网格说明
- Grid Scale 策略说明
- Orderflow 策略说明
- Trend Intraday 重构与开发计划
- Hyperliquid API 相关资料

## 升级说明

- 版本号由 `v0.3.2` 升级至 `v0.3.3`。
- 新增独立仓位报告程序后，可单独运行，不再需要绑定任意策略生命周期。
- 新增/变更的策略配置较多，部署前建议重新核对账户 ID、交易对精度、最小名义价值和 webhook 地址。
