# 仓位接口官方核验 P0 第一批

状态日期：2026-06-08

本批从 P2 仓位剩余队列先处理 12 个 adapter。判断口径：

- 官方/API/SDK/协议可读取仓位，但当前项目没有 parser、签名器、链上账户扫描或 read-only runtime，写 `项目未实现`。
- 当前 adapter/profile 没有稳定仓位接口，或只是路由/壳 profile，不应接 live position runtime，写 `交易所不支持当前仓位接口 runtime`。

## 核验结论

| adapter | 产品线 | 官方结论 | 项目当前状态 | 写法 |
| --- | --- | --- | --- | --- |
| `aark` | Perpetual | Orderly EVM private REST 支持 `GET /v1/positions`，返回 margin ratio、free collateral、position_qty、mark price、liq price、funding、margin_mode 等字段。 | 当前 `positions` 返回 Orderly account audit required。 | 官方支持，项目未实现/未启用。 |
| `modetrade` | Perpetual | ModeTrade 是 Orderly EVM profile，同源 Orderly `GET /v1/positions` 可作为账户仓位来源。 | 当前 `positions` 返回 Orderly account audit required。 | 官方支持，项目未实现/未启用。 |
| `aftermath` | Perpetual | Aftermath Perpetuals SDK 的 account/account objects 返回 positions、available collateral、total equity，并按 account cap / account id 查询。 | 当前只接 public CCXT market/orderbook；private/account runtime disabled。 | 官方支持，项目未实现链上账户仓位读取。 |
| `bybiteu` | Spot/Futures/Perpetual | Bybit V5 position API 支持 `GET /v5/position/list`，category 覆盖 linear、inverse、option，支持 symbol/baseCoin/settleCoin/cursor。 | regional profile 禁用 private REST/WS。 | 官方支持，项目未实现/未启用。 |
| `d8x` | Perpetual | D8X 是链上 perpetual futures engine，仓位读取需要 wallet/contract/indexer 或 SDK 账户状态解析。 | 当前只接 CoinGecko-compatible public REST market data。 | 官方协议支持，项目未实现链上仓位 runtime。 |
| `grvt` | Perpetual/Option | GRVT API docs 有 positions request schema，trading/auth 资料使用 session/account 模型。 | `positions` 是 session-spec-only。 | 官方支持，项目未实现/未启用。 |
| `hibachi` | Perpetual | Hibachi 官方交易/风险资料明确有 positions、cross margin、subaccount isolated-like usage、risk panel 和 liquidation 语义。 | 当前 private account/positions 只做 request-spec/audit boundary。 | 官方支持，项目未实现/未启用。 |
| `lighter` | Perpetual | Lighter SDK/API 的 `AccountApi.account` 返回 account data；账号/sub-account、auth token、WS account updates 都依赖 signer/auth token。 | `positions` 返回 auth token unverified。 | 官方支持，项目未实现/未启用。 |
| `mango_markets` | Margin/Perpetual | Mango v4 在 MangoAccount 中跟踪 PerpPosition，包含 base lots、quote native、funding settlement、market index 等。 | G0 audit-only；不做 Solana account decoding。 | 官方协议支持，项目未实现链上仓位 runtime。 |
| `bsx` | Perpetual | 当前官方 API 文档能核到 orders、trade-history、account/order WS，但未核到稳定 positions endpoint 和 position schema。 | public REST 已接；positions unsupported。 | 交易所不支持当前仓位接口 runtime。 |
| `bitkan` | Spot/Perpetual | 当前未核到稳定官方 BitKan-native exchange API、签名规则和仓位生命周期文档。 | 所有 REST/WS/trading/positions 均 unverified unsupported。 | 交易所不支持当前仓位接口 runtime。 |
| `cod3x` | Perpetual routing | Cod3x 是 perps terminal/routing layer；下游 venue 的 positions 走各自 adapter，不并入 Cod3x-native profile。 | audit-only，不接 Cod3x-native runtime。 | 交易所不支持当前仓位接口 runtime。 |

## 后续实现边界

- Orderly profiles（`aark`、`modetrade`）：先补 account id/key/region/KYC 审计，再实现 signed `GET /v1/positions` parser。
- 链上/SDK profiles（`aftermath`、`d8x`、`mango_markets`）：不要套 REST HMAC；需要账户对象解析、链上确认、slot/reorg/indexer 对账。
- API-key/session profiles（`bybiteu`、`grvt`、`hibachi`、`lighter`）：先做 read-only credential/auth token smoke test，再接 positions parser。
- `bsx`、`bitkan`、`cod3x`：保持 unsupported；出现稳定官方 positions endpoint 后再重核。
