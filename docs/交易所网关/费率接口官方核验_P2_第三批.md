# 费率接口官方核验 P2 第三批

状态日期：2026-06-08

本批收口剩余 15 个 `fee_official_check`。判断口径延续前两批：

- 官方有账户费率 API、交易费率表、VIP fee schedule、maker/taker fee model，且可以作为 `FeeRateSnapshot` 来源的，写 `官方支持，项目未实现/未启用`。
- 只有成交回报中的已成交手续费金额、提现手续费、白标 venue 非稳定配置，或当前 profile 没有可映射成共享 maker/taker 费率 runtime 的，写 `交易所不支持当前费率接口 runtime`。
- 默认费率、零费率占位、未验证 fallback 不能当生产费率；必须显式标注 source，并优先补官方 endpoint、官方费率表或配置型 fee source。

## 核验结论

| adapter | 产品线 | 官方/项目结论 | 项目当前状态 | 写法 |
| --- | --- | --- | --- | --- |
| `bigone` | Spot/Contract | 官方 Spot 有 `GET /viewer/trading_fees`，返回 authenticated user 的 maker/taker fee rate；Contract 有 maker/taker fee schedule。 | 当前 `get_fees_impl` 使用 private/default parser，矩阵没有标准 fee source 证据；默认值不能当生产费率。 | 官方支持，项目未实现/未启用。 |
| `bitmart` | Spot/Futures | 官方 fee 页面有 Spot/Futures VIP maker/taker fee schedule，fee level 每日更新。 | 当前 `get_fees_impl` 返回默认 spot/perp 费率，source 是 `bitmart_default_until_private_fee_live_validated`。 | 官方支持，项目未实现/未启用。 |
| `bitso` | Spot | 官方 `GET /api/v3/fees` 返回每个 book 的 customer maker/taker trading fee。 | `supports_fees=false`，runtime reason `bitso.fees_private_read_not_promoted`。 | 官方支持，项目未实现/未启用。 |
| `btcturk` | Spot | 官方 BtcTurk Global fee structure 给出 Market Maker/Taker tier schedule；当前 adapter 精确 API fee readback 未验证。 | runtime reason `btcturk.fees_unverified`。 | 官方支持，项目未实现/未启用；先按配置型 fee source 处理。 |
| `cex` | Spot | 官方 Spot Trading API 有 `POST /get_my_current_fee`；Legacy V2 也有 `get_myfee` 语义。 | `supports_fees=false`，runtime reason `cex.fees_private_rest_deferred`。 | 官方支持，项目未实现/未启用。 |
| `fmfwio` | Spot/Futures profile | HitBTC-family v3 官方 API 有 `GET /api/3/spot/fee` 和 `GET /api/3/futures/fee`。 | `get_fees` 返回 `fmfwio.get_fees_spec_only`，private runtime 未启用。 | 官方支持，项目未实现/未启用。 |
| `grvt` | Perpetual/Option | 官方 help center 有 active maker/taker fee tier model；account tier 需要 session/account scope。 | `get_fees` 返回 `grvt.fees_session_spec_only`。 | 官方支持，项目未实现/未启用；先按 fee tier/config source 处理。 |
| `hitbtc` | Spot/Margin/Futures profile | HitBTC v3 官方 API 有 Spot/Futures trading commission endpoints。 | `get_fees` 返回 `hitbtc.get_fees_spec_only`，private runtime 未启用。 | 官方支持，项目未实现/未启用。 |
| `latoken` | Spot/Futures profile | 官方 v2 docs 列出 `/v2/auth/trade/fee/{currency}/{quote}` 和 `/v2/trade/fee/{currency}/{quote}`。 | `supports_fees=false`，runtime reason `latoken.fees_unverified`。 | 官方支持，项目未实现/未启用。 |
| `lighter` | Perpetual/Spot boundary | 官方 fee/support 资料说明 standard account maker/taker 为 0，premium/API/HFT 账户可能自定义。 | `get_fees` 返回 `lighter.fees_auth_token_unverified`，signed tx/account runtime 未启用。 | 官方支持，项目未实现/未启用；先按 zero-fee/default + premium override 配置处理。 |
| `novadax` | Spot | 官方 fees/limits 页面给出 VIP maker/taker table；API 成交回报有 fee amount/role，但未核到 account-effective fee-rate endpoint。 | `get_fees` 返回 `novadax.fees_private_read_not_promoted`。 | 官方支持，项目未实现/未启用；先按官方 fee table/config source 处理。 |
| `bittrade` | Spot | 官方 fee 页面有交易所手续费表，但当前 API docs 未核到可读 account-effective maker/taker fee endpoint。 | `get_fees` 返回 `bittrade.get_fees.offline_request_spec_only`，private runtime 不暴露 live readback。 | 交易所不支持当前费率接口 runtime。 |
| `foxbit` | Spot | 当前已核资料没有稳定 trading fee-rate endpoint；成交手续费金额不能等同 maker/taker rate readback。 | `get_fees` 返回 `foxbit.fees_private_read_not_promoted`。 | 交易所不支持当前费率接口 runtime。 |
| `hollaex` | Spot/white-label profile | 官方 demo/API profile 可见 withdrawal fee 和 usertrade fee amount，但未见稳定 maker/taker trading fee-rate endpoint；任意白标 venue 费率不同。 | `get_fees` 返回 `hollaex.get_fees_spec_only`。 | 交易所不支持当前费率接口 runtime。 |
| `ndax` | Spot | 当前 adapter/API 证据只到 public/user API、risk tiers 和 offline private specs，未核到 maker/taker fee readback。 | `get_fees` 返回 `ndax.fees_offline_request_spec_only`。 | 交易所不支持当前费率接口 runtime。 |
