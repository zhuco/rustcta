# 核心交易官方核验 P1 第二批

状态日期：2026-06-08

本批继续核验下单、撤单、查单、open orders、fills、client order id、TIF 和 market/limit 支持。结论只处理核心交易接口；产品线、公共 WebSocket、仓位、账户和费率仍以各自核验文件为准。

## 核验结论

| adapter | 官方核心交易接口 | 订单类型 / TIF | client order id | 当前项目状态 | 结论 |
| --- | --- | --- | --- | --- | --- |
| `hitbtc` | HitBTC v3 支持 `POST /api/3/spot/order`、`DELETE /api/3/spot/order`、`GET /api/3/spot/order`、`GET /api/3/spot/order/{client_order_id}`、`DELETE /api/3/spot/order/{client_order_id}`，并有 history/trade。 | 支持 limit、market、stop/take-profit 系列；支持 GTC/IOC/FOK 等 time_in_force。 | 支持 `client_order_id`，8-32 字符，唯一到订单 inactive 后一段时间。 | 当前 `place_order=不支持`、`cancel_order=不支持`。 | 官方支持 Spot 核心交易，项目未实现；不能写交易所不支持下单/撤单。 |
| `fmfwio` | FMFW.io 是 HitBTC v3 family API，官方接口同样支持 Spot order create/cancel/query/history，并有 margin/futures order family。 | 同 HitBTC v3 family：limit/market/stop/take-profit，GTC/IOC/FOK 等。 | 支持 `client_order_id`。 | 当前 `place_order=不支持`、`cancel_order=不支持`。 | 官方支持核心交易，项目未实现；后续可复用 HitBTC v3 family mapping。 |
| `indodax` | Indodax TAPI/PDF 官方文档支持 `trade`、`cancelOrder`、`openOrders`、`orderHistory`、`tradeHistory`。 | Spot buy/sell；官方 TAPI 按 pair/type/price/amount 语义；具体 TIF 不适用。 | 官方 TAPI 以 exchange order id 撤单，未见 client order id。 | 当前 `place_order=运行`，`cancel_order=不支持`。 | 下单已运行，撤单官方支持但项目未实现；补 cancel runtime 和 parser。 |
| `paymium` | Paymium API 支持 `POST /api/v1/user/orders` 创建订单和订单取消接口，并提供 user orders/history。 | 支持 `LimitOrder`、`MarketOrder`；buy/sell；EUR quote。 | 官方公开页未见通用 client order id。 | 当前 `place_order=不支持`、`cancel_order=不支持`。 | 官方支持 BTC/EUR Spot 核心交易，项目未实现 private trading runtime。 |
| `p2b` | P2B REST API 支持 `POST /api/v2/order/new` 创建订单、`POST /api/v2/order/cancel` 撤单、`POST /api/v2/orders` 查询未成交订单、`POST /api/v2/account/order_history`。 | Spot limit buy/sell；amount/price 必填；market 由已启用 markets 决定。 | 官方 API 以 numeric `orderId` 撤单；未见 client order id。 | 当前 `place_order=不支持`、`cancel_order=不支持`。 | 官方支持核心交易，项目未实现 signed private runtime。 |
| `coinspot` | CoinSpot API/V2 官方页面列出 place buy/sell order、cancel buy/sell order、market buy/sell/cancel 等私有交易接口。 | CoinSpot 支持 instant buy/sell、market orders、advanced buy stop/limit；API V2 有 market buy/sell 和 cancel。 | 官方公开页未见统一 client order id。 | 当前 `place_order=运行`，`cancel_order=-`。 | 下单已有运行证据，撤单官方支持但矩阵缺实现证据；补 cancel runtime/fixture 或修正 mapping。 |
| `myokx` | MyOKX regional profile 使用 OKX V5 Trade API：`POST /api/v5/trade/order`、`POST /api/v5/trade/cancel-order`、batch order/cancel、orders-pending/history。 | 支持 limit、market、post_only、fok、ioc、optimal_limit_ioc 等；Trade permission required。 | 支持 `clOrdId`；撤单可用 `ordId` 或 `clOrdId`。 | 当前 regional profile 禁用 private trading runtime。 | 官方支持，但项目未实现/未启用区域 private trading；不能写交易所不支持。 |
| `okxus` | OKX US profile 复用 OKX V5 Trade API 语义；区域 eligibility、API 域名和产品 scope 需要单独审计。 | 同 OKX V5；Spot 下单至少支持 limit/market 等基础订单类型。 | 支持 `clOrdId`。 | 当前 OKX US profile 禁用 private REST/WS 和真实下单撤单。 | 官方/区域资料有交易接口线索，项目未实现并需区域凭证审计；不能写交易所不支持。 |
| `weex` | WEEX Spot V3 支持 `POST /api/v3/order` 和 Spot cancel；Contract API 支持 V2/V3 place/cancel、batch cancel、cancel all。 | Spot 支持 LIMIT/MARKET，TIF 支持 GTC/IOC/FOK；Contract 支持 normal/post-only/FOK/IOC、market/limit。 | Spot `newClientOrderId`；Contract `client_oid` 或 `origClientOrderId`。 | 当前矩阵 `place_order=-`、`cancel_order=-`，但 adapter 文档显示已有 private order lifecycle 语义。 | 官方支持 Spot/Perp 核心交易；当前矩阵需补/修 mapping 或补缺失 runtime evidence。 |

## 下一步实现建议

1. `hitbtc` 和 `fmfwio` 共享 v3 family trade mapping，可先补 Spot order/cancel/query/open/fills request specs。
2. `indodax`、`coinspot` 已有下单运行证据，应优先补撤单，避免半套交易 lifecycle。
3. `myokx`、`okxus` 先补 private REST request specs 和区域凭证 scope guard，未过审计前不启用 runtime。
4. `weex` 先核对 endpoint mapping 为什么矩阵仍显示 `-`；若代码已实现，修 mapping evidence；若缺口真实存在，补 Spot/Perp order lifecycle specs。
