## 每个交易所官方核验要填的字段

| 字段 | 说明 |
| --- | --- |
| 官方文档 URL | 必须是交易所官方域名或官方 GitHub/API docs。 |
| 产品线结论 | Spot、Margin、USDT 永续、币本位、期权分别写 `官方支持` 或 `交易所不支持`。 |
| 公共 WS 订阅方式 | URL path、JSON subscribe、combined stream、Socket.IO、单连接订阅限制。 |
| 订单簿 channel | L1/BBO、partial depth、full depth delta、snapshot。 |
| 推流间隔 | 例如 10ms、20ms、50ms、100ms、250ms、1000ms；官方只写 real-time 也要原样记录，并单独标是否有 10ms/20ms L1/BBO。 |
| 档位 | 1、5、10、20、50、100、200、500 或全量增量。 |
| sequence/checksum | 更新 ID、prev ID、CRC/checksum、丢包重建规则。 |
| REST snapshot | endpoint、depth limit、delta buffer 规则。 |
| 私有 WS | orders、fills、balances、positions、auth、续期、断线 REST 兜底。 |
| 写接口 | 下单、撤单、全撤、批量、改单、client order id、TIF、reduce-only。 |

## 推荐工作顺序

1. 先按 [交易所网关补全任务清单](交易所网关补全任务清单.md) 做 P0 公共订单簿 WebSocket，因为跨所套利没有低延迟盘口无法上线。
2. 再做 P0 产品线项目未实现任务，优先 Binance USD-M、OKX Swap、Bitget Futures、Gate.io Perpetual、MEXC Contract。
3. 再做 P1 已有 WS 交易所，把推流间隔、档位、sequence/checksum 结构化进 endpoint mapping。
4. 产品线缺口、公共 WS 细项、核心交易缺口、仓位接口、账户/余额接口、费率接口和高级订单能力已经全部转入任务或 unsupported 标记；当前不再保留剩余官方核验项。
5. 最后补私有 WS、账户/费率和批量/改单等交易增强能力。
