# 高级订单能力官方核验 P3 第四批

状态日期：2026-06-08

本批继续收口 `advanced_trading_official_check` 中本地 adapter 文档和 endpoint mapping 已经能给出结论的 12 个 adapter。口径延续前三批：

- 批量、改单、OCO/OTO/order-list 中只要有一部分已映射或 spec-only，就写 `advanced_trading`，并在必补内容里写清缺失子能力。
- 当前 profile 没有可进入共享高级订单 runtime 的 batch/amend/order-list 语义，或 private runtime 本身仍 disabled，则写 `advanced_trading_unsupported`。

## 核验结论

| adapter | 产品线 | 官方/项目结论 | 项目当前状态 | 写法 |
| --- | --- | --- | --- | --- |
| `alpaca` | Spot | Alpaca Broker crypto order API 是单订单；当前 adapter 支持 account-wide cancel-all，并把批量撤单声明为网关顺序组合。 | batch place unsupported；batch cancel composed sequential；amend/order-list 未进入共享模型。 | 补矩阵识别 composed batch cancel；batch place、amend、order-list 写 unsupported 边界。 |
| `apollox_dex` | Perpetual | ApolloX DEX V1 Futures profile 有 native batch place spec；batch cancel 未启用 gateway-safe mapping。 | `batch_place_orders` 为 spec_only native batch，`batch_cancel_orders` unsupported；private writes 仍是 spec-only。 | 补 batch place runtime/request spec 验收；batch cancel、amend/order-list 写 unsupported 边界。 |
| `aster` | Perpetual | Aster V3 当前 profile 已接普通 order/cancel/cancel-all，但 endpoint mapping 没有 batch endpoint，且 order-list/amend 显式 unsupported。 | batch place/cancel 缺失；`order_list`、`amend_order` 在 unsupported 列表。 | 写 `交易所不支持当前高级订单能力 runtime`；普通下单/撤单继续保留。 |
| `bequant` | Spot | Bequant Spot profile 支持普通 order/cancel/cancel-all，但没有映射 native batch create、batch cancel-by-list 或 generic amend。 | endpoint mapping 明确 batch place/cancel unsupported、amend_order unsupported。 | 写 `交易所不支持当前高级订单能力 runtime`；cancel-all 不纳入本高级订单口径。 |
| `bigone` | Spot/Perpetual | BigONE 已实现 Spot native batch place、contract native batch cancel，部分 batch 走顺序组合；amend 和 OCO/order-list 不支持。 | batch place/cancel 运行；adapter 文档明确 amend order、order-list/OCO unsupported。 | 保留已实现 batch；amend、order-list/OCO 写 unsupported 边界。 |
| `binance` | Spot | Binance Spot adapter 已实现 keep-priority amend 和 OCO order-list；当前缺口只剩 batch place/cancel。 | `amend_order=运行`、`place_order_list=运行`；batch place/cancel unsupported。 | 写清 batch place/cancel 当前不支持；不影响已实现 amend/OCO。 |
| `binancecoinm` | COIN-M Futures/Perpetual | Binance COIN-M adapter 已实现 futures amend；Spot OCO/order-list 不适用；batch place/cancel 尚未实现。 | `amend_order=运行`；place_order_list、batch place/cancel unsupported。 | 批量接口写项目未实现/未启用；order-list 写当前 COIN-M profile 不支持。 |
| `bit2c` | Spot | Bit2C Pro 当前官方/API profile 只保留普通私有 REST request-spec；没有共享 batch/amend/order-list runtime。 | adapter 文档明确 batch place/cancel、cancel-all、amend、order lists unsupported。 | 写 `交易所不支持当前高级订单能力 runtime`。 |
| `bitbank` | Spot | bitbank Spot 有 native same-pair batch cancel；无 standard amend endpoint，也没有 batch place/order-list。 | `batch_cancel_orders=映射`；batch place unsupported；amend unsupported。 | 补矩阵识别 batch cancel；batch place、amend、order-list 写 unsupported 边界。 |
| `bitbns` | Spot/Margin boundary | Bitbns private REST 仍 disabled；当前 adapter 不启用 place/cancel/cancel-all/amend/order-list/batch。 | public REST scan-only；Unsupported Boundaries 明确所有 private writes 和 batch disabled。 | 写 `交易所不支持当前高级订单能力 runtime`；普通交易另按核心交易任务补 private runtime。 |
| `bitfinex` | Spot/Margin/Perpetual | Bitfinex 已实现 native batch cancel/cancel-all；batch place multi-op 和 amend 仍未映射到共享语义。 | `batch_cancel_orders=运行`；batch place unsupported；amend unsupported。 | 保留 batch cancel；补/记录 batch place 和 amend unsupported 边界。 |
| `bitflyer` | Spot/Margin boundary | bitFlyer 当前 mapping 声明 batch place/cancel 为网关顺序组合；官方没有共享 order-list/OCO 或 in-place amend 映射。 | top-level batch composed；矩阵未识别 batch 字段。 | 补矩阵识别 composed batch；amend/order-list 写 unsupported 边界。 |
