# 核心交易官方核验 P3 第四批

状态日期：2026-06-08

本批收口剩余核心交易官方核验队列。结论分两类：

- 官方或协议资料支持下单/撤单，但当前项目没有签名器、账户审计、链上交易构造或私有 runtime，写 `项目未实现`。
- 当前 adapter/profile 没有稳定的交易所网关交易 API，或交易 runtime 已由另一个 adapter 承接，写 `交易所不支持当前交易/私有接口 runtime`。

## 核验结论

| adapter | 产品线 | 官方结论 | 项目当前状态 | 写法 |
| --- | --- | --- | --- | --- |
| `aark` | Perpetual | Aark Perpetual Mode 走 Orderly infrastructure；Orderly signed account endpoints 支持订单生命周期，但 Aark profile 的账户 onboarding、key 权限和区域限制还未审计。 | `place_order`/`cancel_order` 返回 Orderly Ed25519 account audit required。 | 官方支持，项目未实现/未启用。 |
| `modetrade` | Perpetual | Mode Trade 是 Orderly EVM profile；Orderly signed account endpoints 支持订单生命周期，但 ModeTrade profile 语义未单独验证。 | `place_order`/`cancel_order` 返回 Orderly Ed25519 account audit required。 | 官方支持，项目未实现/未启用。 |
| `aftermath` | Perpetual | Aftermath Perpetuals 是 Sui on-chain protocol，交易需要 Sui transaction build/sign/submit 和账户授权。 | 当前只接 public CCXT markets/orderbook；私有读写均 `Unsupported`。 | 官方协议可交易，项目未实现链上交易 runtime。 |
| `bitbns` | Spot/Margin | 官方 SDK/site 提供 API Trading、私有签名和交易接口线索。 | 当前只做 public REST scan；私有 REST 和下单/撤单禁用。 | 官方支持，项目未实现/未启用。 |
| `bitteam` | Spot | 官方 OpenAPI 列出 `ordercreate`、`cancelorder`、open orders 和 user trades。 | public REST 已接；private endpoints 是 request-spec-only。 | 官方支持，项目未实现/未启用。 |
| `bsx` | Perpetual | 官方 REST 支持 `POST /orders`、`DELETE /orders/{order_id}`，订单需要 account headers 和 EIP-712 order signature。 | public REST 已接；private write request-spec-only。 | 官方支持，项目未实现/未启用。 |
| `d8x` | Perpetual | D8X trading access 走 Futures Node SDK 或 smart contracts，需要 EVM wallet signer 和链上交易对账。 | 当前只接 CoinGecko-compatible public REST market data。 | 官方协议可交易，项目未实现链上交易 runtime。 |

| `hibachi` | Perpetual | 官方 trade REST 支持 `POST/DELETE/PATCH /trade/order` 和批量 orders endpoints。 | public market REST 已接；write request specs 和签名向量离线。 | 官方支持，项目未实现/未启用。 |
| `mango_markets` | Margin/Perpetual | Mango v4 SDK 支持 wallet/MangoAccount 驱动的订单交易；需要 Solana wallet-signed transactions。 | G0 audit-only；不构造或提交交易。 | 官方协议可交易，项目未实现链上交易 runtime。 |
| `bitkan` | Spot/Perpetual | 当前未核到稳定官方 BitKan-native exchange trading API、签名规则和订单生命周期文档。 | 当前 adapter 所有 REST/WS/trading 均 unverified unsupported。 | 交易所不支持当前交易/私有接口 runtime。 |
| `cod3x` | Perpetual routing | Cod3x 是 AI perps terminal/routing layer，未建立稳定 Cod3x-native exchange API；下游 Hyperliquid/GMX/Lighter 走各自 adapter。 | audit-only，不接 Cod3x-native trading runtime。 | 交易所不支持当前交易/私有接口 runtime。 |
| `derive_chain_perps` | Perpetual profile | 本 profile 只记录 Derive Chain settlement/profile boundary；Derive Exchange API runtime 归 `derive` adapter。 | 不重复接 Derive REST/WS/order runtime。 | 交易所不支持当前 profile 交易/私有接口 runtime。 |
| `equation` | Perpetual | 官方资料是 Equation perpetual protocol / Arbitrum contract examples；未给稳定 exchange-gateway 交易 API。 | audit-only，不接 REST/GraphQL/WS/EVM signing runtime。 | 交易所不支持当前交易/私有接口 runtime。 |

## 后续实现边界

- `aark`、`modetrade`：先做 Orderly/Aark or ModeTrade account audit、Ed25519 signing vectors、private read-only reconciliation，再开放 order write dry-run。
- `aftermath`、`d8x`、`mango_markets`：先设计链上交易构造、签名、submit、reorg/slot/nonce、indexer 对账，不要套 HMAC REST 模型。
- `bitbns`、`bitteam`、`bsx`、`grvt`、`hibachi`：先补私有 request spec、签名器、只读账号 smoke test、order/cancel parser 和 dry-run guard。
- `bitkan`、`cod3x`、`derive_chain_perps`、`equation`：保持 unsupported；只有出现稳定官方 gateway API 或明确 profile runtime owner 后再重核。
