# 高级订单能力官方核验 P0 第一批

状态日期：2026-06-08

本批先收口 `advanced_trading_official_check` 中已有强证据的 5 个 adapter。高级订单能力口径包括：

- `amend_order`：改单、replace、modify。
- `place_order_list`：OCO/OTO/OTOCO 或订单列表语义。
- `batch_place_orders` / `batch_cancel_orders`：原生批量下单/批量撤单。

判断口径：

- 官方或 adapter 文档已确认支持批量、改单或等价 endpoint，但当前项目只到 request-spec/session-spec/signing 未实现，写 `官方支持，项目未实现/未启用`。
- 官方/adapter 资料明确没有原生批量、改单或 OCO/OTO endpoint，或当前 profile 不承接该高级订单语义，写 `交易所不支持当前高级订单能力 runtime`。
- 只支持普通下单/撤单不等于支持高级订单；普通下单能力仍归核心交易任务。

## 核验结论

| adapter | 产品线 | 官方/项目结论 | 项目当前状态 | 写法 |
| --- | --- | --- | --- | --- |
| `grvt` | Perpetual/Option | 官方 trading/auth 资料支持 create/cancel/bulk orders，adapter mapping 也列出 `/v2/bulk_orders` 和 amend/session-spec 边界。 | `bulk_orders`、`amend_order` 仍是 session-spec-only，缺 EIP-712/external signer 和 live session runtime。 | 官方支持，项目未实现/未启用。 |
| `hibachi` | Perpetual | 官方 trade REST 支持 `PATCH /trade/order` 和 `POST/DELETE /trade/orders` 批量 endpoints。 | `amend_order`、`batch_orders` 仍是 request-spec-only，缺 account/signing runtime。 | 官方支持，项目未实现/未启用。 |
| `lighter` | Perpetual | 官方 API Trading 资料支持 `/batchOrders`，项目 mapping 也列 `batch_tx`、`batch_cancel` 和 `amend_order`。 | `send_tx_batch_signing_unverified`、`send_tx_modify_signing_unverified`，缺 SDK-compatible signed tx runtime。 | 官方支持，项目未实现/未启用。 |
| `apex` | Perpetual | 已审计官方资料未找到 native amend、batch place/cancel、OCO/OTO order-list endpoint。 | endpoint mapping 明确 `amend_order`、`order_list`、`batch_place_orders`、`batch_cancel_orders` unsupported。 | 交易所不支持当前高级订单能力 runtime。 |
| `bullish` | Spot/Perpetual | 已审计官方 REST spec 未找到 native batch place/cancel 和 OCO/OTO order-list endpoint。 | endpoint mapping 明确 `batch_place_orders`、`batch_cancel_orders`、`order_list` unsupported；普通 amend 另有 request-spec。 | 交易所不支持当前高级订单能力 runtime。 |
