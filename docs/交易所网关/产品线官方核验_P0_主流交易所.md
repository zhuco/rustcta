# 产品线官方核验 P0 主流交易所

状态日期：2026-06-08

本文件核验主流交易所官方支持的产品线，并和当前项目 adapter 声明对齐。这里的结论只解决“交易所官方是否支持某产品线”和“项目当前是否接入”；接口级别细节仍看 [交易所功能盘点矩阵](交易所功能盘点矩阵.md) 和 [官方能力核验台账](官方能力核验台账.md)。

## 核验口径

- 官方支持但项目没有 adapter 或 endpoint mapping：写 `项目未实现`。
- 官方 API 产品线不存在，或该 adapter 对应的是独立产品 API：写 `交易所不支持`，并注明口径。
- 同品牌多产品线拆 adapter 时，不把一个 adapter 缺口误判成品牌缺口。
- 现货和合约分离的交易所，优先保留现有 adapter 边界，例如 `kucoin` / `kucoinfutures`、`binance` / `binancecoinm`。

## 产品线矩阵

| 交易所/adapter | 官方产品线结论 | 当前项目声明 | 结论和下一步 |
| --- | --- | --- | --- |
| `binance` | Binance 官方有 Spot、USD-M Futures、COIN-M Futures、Options。 | `binance` 只声明 Spot；`binancecoinm` 声明 COIN-M futures/perpetual。 | `binance` 缺合约不是交易所不支持，而是项目未实现 USD-M/Options；COIN-M 已拆到 `binancecoinm`。 |
| `binancecoinm` | Binance COIN-M 是独立衍生品 API，不是现货 API。 | futures/perpetual。 | 对 `binancecoinm` adapter 写 `交易所不支持现货`；现货走 `binance`。 |
| `binanceus` | Binance.US 官方 API 资料只覆盖 Spot/Wallet/User Data Stream。 | Spot。 | 当前按 `交易所不支持合约` 处理；如 Binance.US 后续开放合约，需要重新核验。 |
| `okx` | OKX 官方支持 Spot、Margin、Swap、Futures、Options，并用 `instType` 区分。 | 当前 `okx` 只声明 Spot。 | `okx` 合约/期权是项目未实现，不是交易所不支持；应补 Swap/Futures/Option 或拆 adapter。 |
| `okxus` / `myokx` | 区域站点当前项目按 Spot 适配；官方全球 OKX 多产品线不能直接套到区域站。 | Spot。 | 不能把全球 OKX 的合约能力直接写给区域 adapter；区域产品线需要单独官方核验。 |
| `bybit` | V5 API 统一 Spot、Linear、Inverse、Option。 | Spot + futures/perpetual。 | 产品线覆盖方向正确；下一步重点补 WS 细项和缺失写接口。 |
| `bitget` | 官方 API 顶部产品线包含 Spot、Futures、Margin 等。 | 当前 `bitget` 只声明 Spot。 | 合约是项目未实现，不是交易所不支持；应补 Futures adapter/endpoint mapping。 |
| `gateio` | Gate API v4 覆盖 Spot/Margin、Perpetual Futures、Delivery Futures、Options。 | 当前 `gateio` 只声明 Spot。 | 合约和期权是项目未实现，不是交易所不支持。 |
| `mexc` | 官方有 Spot API 和 Contract API。 | 当前 `mexc` 只声明 Spot。 | 合约是项目未实现，不是交易所不支持。 |
| `kucoin` | KuCoin 官方有 Spot API；KuCoin Futures 是独立 futures API。 | `kucoin` Spot；`kucoinfutures` Perpetual。 | adapter 边界合理；`kucoin` 不需要混入 futures，合约走 `kucoinfutures`。 |
| `kucoinfutures` | KuCoin Futures 是独立合约 API。 | Perpetual。 | 对 `kucoinfutures` adapter 写 `交易所不支持现货`；现货走 `kucoin`。 |
| `htx` / `huobi` | 官方有 Spot API 和 USDT-Margined Swap/Futures API。 | Spot + perpetual。 | 产品线方向正确；下一步补 WS 低延迟盘口细项。 |
| `lbank` | 官方资料覆盖 Spot 和合约/永续 API。 | Spot + perpetual。 | 产品线方向正确；下一步补合约 WS/私有接口细节。 |
| `coinw` | 官方资料覆盖 Spot 和 Futures/Perpetual API。 | Spot + perpetual。 | 产品线方向正确；下一步补结构化 WS channel、档位和 sequence。 |
| `coinbase` | Coinbase Advanced Trade 覆盖 Spot；Coinbase International Exchange/INTX 覆盖 perpetual futures。 | Spot + perpetual。 | 产品线方向正确，但要区分 Advanced Trade 与 INTX；不要和 Coinbase Exchange 混用。 |
| `coinbaseexchange` | Coinbase Exchange API 是现货交易所 API。 | Spot。 | 对 `coinbaseexchange` adapter 写 `交易所不支持合约`；合约/永续走 `coinbase` 的 INTX 路径。 |
| `kraken` | Kraken Spot API 与 Kraken Futures API 都存在，官方产品线分离。 | Spot + perpetual。 | 项目 `kraken` 当前同时声明 Spot/perp，需要确认是否应继续合并或拆清楚 futures 边界。 |
| `krakenfutures` | Kraken Futures 是独立 futures API。 | Perpetual。 | 对 `krakenfutures` adapter 写 `交易所不支持现货`；现货走 `kraken`。 |

## 明确的项目未实现

这些不是交易所不支持，而是当前项目还没有完整接：

| 交易所 | 官方支持 | 当前缺口 |
| --- | --- | --- |
| Binance | USD-M Futures、Options | 当前没有独立 `binanceusdm` / `binanceoption` adapter；`binance` 只接 Spot。 |
| OKX | Swap、Futures、Options | 当前 `okx` 只接 Spot。 |
| Bitget | Futures | 当前 `bitget` 只接 Spot。 |
| Gate.io | Perpetual Futures、Delivery Futures、Options | 当前 `gateio` 只接 Spot。 |
| MEXC | Contract/Futures | 当前 `mexc` 只接 Spot。 |

## 明确的 adapter 口径交易所不支持

这些结论是按 adapter 对应 API 产品线写，不代表同品牌所有产品都没有：

| adapter | 写法 | 说明 |
| --- | --- | --- |
| `binancecoinm` | `交易所不支持现货` | COIN-M adapter 只对应 Binance COIN-M derivatives API。 |
| `kucoinfutures` | `交易所不支持现货` | KuCoin Futures adapter 只对应 futures API。 |
| `krakenfutures` | `交易所不支持现货` | Kraken Futures adapter 只对应 futures API。 |
| `coinbaseexchange` | `交易所不支持合约` | Coinbase Exchange API 是 spot exchange API。 |
| `binanceus` | `交易所不支持合约` | 本轮官方 API 资料只见 Spot；后续如官方开放合约需重核。 |
