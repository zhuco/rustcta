# 高级订单能力官方核验 P2 第三批

状态日期：2026-06-08

本批继续收口 `advanced_trading_official_check` 中本地官方归档和 endpoint mapping 证据已经明确的 8 个 adapter。高级订单能力口径仍限定为：

- `amend_order`：改单、replace、modify。
- `place_order_list`：OCO/OTO/OTOCO 或订单列表语义。
- `batch_place_orders` / `batch_cancel_orders`：原生或网关声明的批量下单/批量撤单。

本批特别区分两类情况：

- 交易所有批量、改单或等价能力，但矩阵还没有识别，或仍需补清楚 OCO/OTO 等子能力边界：写 `advanced_trading`。
- 当前 profile 没有可进入共享高级订单 runtime 的能力：写 `advanced_trading_unsupported`。

## 核验结论

| adapter | 产品线 | 官方/项目结论 | 项目当前状态 | 写法 |
| --- | --- | --- | --- | --- |
| `bsx` | Perpetual | BSX 普通下单/撤单需要 account headers 和 EIP-712 order signature；当前未验证可无损进入共享 batch/amend/order-list runtime。 | private write、batch、cancel-all 是 request-spec-only 或 unsupported runtime；batch place/cancel reason 指向 EIP-712/cancel-v2 语义审计。 | 写 `交易所不支持当前高级订单能力 runtime`；普通下单/撤单继续按核心交易任务补 signer。 |
| `binanceus` | Spot | Binance.US 文档有 cancel-replace 和 legacy OCO 线索，但当前 adapter 不把它们映射成 Binance.com keep-priority amend 或共享 order-list 请求模型；batch place/cancel 也未实现。 | `amend_order`、`place_order_list` 显式 unsupported；batch place/cancel unsupported；cancel-all native。 | 补/确认 Binance.US 专用 cancel-replace/OCO 适配边界；不能直接复用 Binance.com 高级订单模型。 |
| `backpack` | Spot/Perp | Backpack 已有 native batch place；batch cancel 是顺序组合；OCO/OTO order-list 和 amend 未见稳定原生 API。 | batch place native、batch cancel composed；`amend_order` 和 OCO/OTO 返回 Unsupported。 | 矩阵/任务写清 batch 已支持，缺失的 amend/order-list 写 `交易所不支持`。 |
| `xt` | Spot/Perpetual | XT Spot/Perpetual 已有 native batch place/cancel 和 amend endpoints；未验证 Binance-style OCO/OTO order-list 等价接口。 | `amend_order`、`batch_place_orders`、`batch_cancel_orders` 均有 mapping；`place_order_list` unsupported。 | 补矩阵识别 batch/amend；order-list/OCO/OTO 写 `交易所不支持`。 |
| `ascendex` | Spot/Perpetual | AscendEX Spot/Futures batch place/cancel native，单批最多 10；amend、order-list、stop orders 等高级原生功能未进入共享模型。 | endpoint mapping 有 Spot/Futures batch paths；adapter 文档明确 amend/order-list unsupported gap。 | 补矩阵识别 batch；amend/order-list 写 `交易所不支持`。 |
| `biconomy` | Spot | Biconomy 当前 adapter 已有 gateway-composed batch place/cancel，不声明 native atomic batch；amend/OCO 未进入共享模型。 | capabilities v2 声明 composed non-atomic batch place/cancel，partial failure；矩阵仍把 batch 字段列入缺口。 | 修矩阵/alias 识别 composed batch；amend/OCO 作为 unsupported 子能力边界。 |
| `coindcx` | Spot/Futures | CoinDCX Spot 支持 native batch place/cancel；Spot/Futures amend endpoints 已映射；Futures batch 和 Binance-style OCO/OTO order-list 不支持。 | batch place/cancel request specs 已实现；amend 映射已记录；queue 仍缺 amend/order-list 证据。 | 补矩阵识别 amend/batch；order-list 写 `交易所不支持`，Futures batch 维持 unsupported。 |
| `latoken` | Spot/Futures profile | LATOKEN 核心交易资料支持 placeBulk/batch 线索，但当前 private runtime 仍是 request-spec-only；amend、dead-man/cancel-all-after 未找到可无损映射端点。 | batch place/cancel、place/cancel/cancel-all 均 request-spec-only/unsupported runtime；amend/order-list 未映射。 | 补 batch private runtime 或保留 request-spec 边界；amend/order-list 写 `交易所不支持`。 |
