# 核心交易官方核验 P2 第三批

状态日期：2026-06-08

本批继续核验下单、撤单、查单、open orders、fills、client order id、TIF 和 market/limit 支持。结论只处理核心交易接口；产品线、公共 WebSocket、仓位、账户和费率仍以各自核验文件为准。

## 核验结论

| adapter | 官方核心交易接口 | 订单类型 / TIF | client order id | 当前项目状态 | 结论 |
| --- | --- | --- | --- | --- | --- |
| `dydx` | dYdX Chain 官方 Python client 示例覆盖 placing、replacing、canceling orders；Indexer 可查 subaccount orders、order、fills/positions。 | v4 链上订单由 Node transaction/SDK 构造；订单类型、TIF、reduce-only 等应按 dYdX Chain client/validator 交易语义建模。 | 需要按 dYdX v4 order id / client metadata 语义单独映射；不能按普通 REST `client_order_id` 处理。 | 当前 private writes unsupported；只做 Indexer public/private readback。 | 官方支持链上核心交易，但项目未实现 Node wallet transaction signing；不能写交易所不支持下单/撤单。 |
| `lighter` | Lighter API Trading 文档说明 Trading REST `/orders`、`/batchOrders`，WebSocket `orderUpdates`、`fills`，支持 place/cancel single or batch orders。 | 示例包含 limit order、GTC、leverage；还需要按 SDK/`sendTx` 签名交易语义核验 market/post-only/reduce-only。 | 示例使用 client-generated id 做幂等。 | 当前 `place_order=不支持`、batch/spec only；缺 SDK-compatible signed tx。 | 官方支持核心交易，项目未实现 signed tx/API-key private key runtime。 |
| `wavesexchange` | WX Network matcher API 支持 Place Limit Order、Place Market Order、Cancel Order、Cancel All、Get Order Status、Order History、Tradable Balance。 | Limit、Market、Stop-limit；订单最大 30 天；签名是 Waves order/public-key signature，不是 HMAC API key。 | 使用 matcher `orderId`；未见通用 client order id。 | 当前 signed order placement/cancel unsupported。 | 官方支持核心交易，项目未实现 Waves signed order/cancel runtime。 |
| `hollaex` | HollaEx API 支持 `POST /v2/order`、`DELETE /v2/order`、`DELETE /v2/order/all`、`GET /v2/order`，也有 admin order API。 | User order 支持 limit/market，`meta.post_only`，可选 stop price；具体 TIF 未见统一字段。 | 返回 HollaEx order `id`；未见通用 client order id。 | 当前 private trading/readback offline only，runtime unsupported。 | 官方 demo/API profile 支持核心交易，项目未实现 private runtime；白标 venue 需单独核验。 |
| `latoken` | LATOKEN v2 官方 API 有 `/v2/auth/order/place`、`/v2/auth/order/cancel`、`/v2/auth/order/placeBulk`，并有 API 权限 `PLACE_ORDER`、`CANCEL_ORDER`。 | Spot order 支持 side、condition 示例含 `GTC`；bulk 1-50 条。 | 公开文档主要以 LATOKEN order id 处理；client id 需进一步核验。 | 当前 private REST live read/write unsupported，仅 request-spec/signing fixture。 | 官方支持核心交易，项目未实现 private REST runtime 和 live-dry-run 验证。 |
| `yobit` | YoBit Trade API 支持 `Trade`、`ActiveOrders`、`OrderInfo`、`CancelOrder`、`TradeHistory`，Trade API 明确用于创建和取消订单。 | `Trade` 参数是 pair、type、rate、amount；偏现货限价 buy/sell；未见 market/TIF。 | 使用 numeric `order_id`；未见 client order id。 | 当前 private REST request-spec-only，runtime unsupported。 | 官方支持 Spot 核心交易，项目未实现 private runtime。 |
| `zebpay` | ZebPay Trade API 支持 `POST /orders` 新建 limit order、`DELETE /orders/{orderId}` 撤单、`DELETE /orders/CancelAll`、`GET /orders`、`GET /orders/{orderId}/fills`。 | Spot limit order；side 示例 `bid`，`tradeType`/platform 字段需映射；Futures 另属项目未实现产品线。 | 使用 `orderId`；未见通用 client order id。 | 当前 private REST/trading request-spec-only，runtime unsupported。 | 官方支持核心交易，项目未实现 bearer token/onboarding/private runtime。 |
| `zeta_markets` | 历史 Zeta 是 Solana CLOB/perpetual trading venue，但官方文档说明 Zeta Markets 已于 2025-05 停止运营；Final Epoch 说明 2025-05-01 04:00 UTC 后 trading disabled。 | 历史 SDK/程序交易不再作为当前 runtime 承接；不应补 live order runtime。 | 不适用。 | 当前 adapter scan-only，place/cancel unsupported。 | 当前交易所不支持当前交易/私有接口 runtime；保留 legacy public scan，不补下单/撤单。 |

## 下一步实现建议

1. `dydx`、`lighter`、`wavesexchange` 属于签名模型特殊的链上/zk/SDK 交易接口，先补 signing boundary 和 dry-run-safe request construction，不要直接按 HMAC REST adapter 套。
2. `hollaex`、`latoken`、`yobit`、`zebpay` 属于官方支持现货私有交易但当前 runtime 未启用，先补 read-only/private auth smoke，再补小额 dry-run gate。
3. `zeta_markets` 写成 `交易所不支持当前交易/私有接口 runtime`，不要排入 live trading 补全任务。
