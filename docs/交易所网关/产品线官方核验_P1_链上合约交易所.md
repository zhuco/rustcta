# 产品线官方核验 P1 链上合约交易所

状态日期：2026-06-08

本文件核验一批高优先级链上永续、合约 DEX、Orderly 前端/profile 和已停运 venue。结论只解决产品线和 adapter 边界；项目当前实现状态仍以 [交易所功能盘点矩阵](交易所功能盘点矩阵.md) 为准。

## 核验口径

- `交易所不支持现货`：官方产品/API 当前没有可映射到共享 `Spot` 的交易面，或该 adapter 明确只对应独立合约/永续 profile。
- `项目未实现`：官方有 spot/OpenBook/spot market 线索，但当前 adapter 没有声明或实现。
- `停运/不接入`：官方说明 venue 已停止运营，交易、私有接口和 WS 不应补 runtime。
- 对 Orderly、Derive、Aftermath 这类“品牌/协议有多个产品，但当前 adapter 只接一个 profile”的场景，结论按 adapter 对应 API/profile 写，避免把同品牌其他产品线误判成当前 adapter 能力。

## 产品线矩阵

| adapter | 官方产品线结论 | 当前项目声明 | 结论和下一步 |
| --- | --- | --- | --- |
| `cod3x` | Cod3x 官方文档描述为 AI perps trading terminal，当前执行路径是 Hyperliquid、GMX V2 等下游 venue。未见 Cod3x-native 现货 exchange API。 | `perpetual`，runtime REST/WS/signing unsupported。 | 对 `cod3x` adapter 写 `交易所不支持现货`；不要把下游 venue API 复制成 Cod3x API。 |
| `derive_chain_perps` | Derive 平台整体支持 options、perpetuals、spot；但本 adapter 是 Derive Chain perpetual settlement/profile，不重复 `derive` adapter。 | `perpetual`，runtime REST/WS/signing unsupported_unverified。 | 对 `derive_chain_perps` profile 写 `交易所不支持现货`；Derive 平台其他产品由 `derive` adapter 或后续任务单独处理。 |
| `equation` | Equation V3 官方定位为 perpetual protocol / Perpetual DEX。未见现货 API。 | `perpetual`，runtime API unsupported_unverified。 | 写 `交易所不支持现货`；公共 REST/WS 和 EVM signing 仍需稳定 API 后再接。 |
| `zeta_markets` | 官方文档声明 Zeta Markets 已于 2025-05 停止运营；历史产品是 Solana perpetual futures/orderbook venue。 | `perpetual`；公共 REST 历史 fixture，WS/trading unsupported。 | 写 `交易所不支持现货`；并写 `交易所不支持当前交易/私有接口/公共 WS runtime`，只保留 legacy scan。 |
| `aark` | Aark Perpetual Mode 是 Orderly-powered decentralized perpetual futures market。未见 Aark spot exchange API。 | `perpetual`；只解析 Orderly public info。 | 写 `交易所不支持现货`；私有读写和 WS runtime 继续等 Aark/Orderly account audit。 |
| `aftermath` | Aftermath Perpetuals 是 Sui fully onchain perpetual futures exchange。 | `perpetual`；公共 CCXT markets/orderbook native。 | 对 `aftermath` perps adapter 写 `交易所不支持现货`；钱包交易/私有账户仍需 Sui transaction boundary。 |
| `bsx` | BSX 官方 API intro/文档描述为 decentralized spot and perpetuals exchange；当前 adapter 只做 Base perpetuals。 | `perpetual`；公共 products/orderbook native。 | 不能写交易所不支持现货；写 `项目未实现 Spot` 或“当前 `bsx` adapter 未接 spot”。 |
| `d8x` | D8X 官方定位为 institutional-grade perpetual futures engine。未见现货交易产品。 | `perpetual`；公共 market-data REST native。 | 写 `交易所不支持现货`；交易写入需要 EVM signer/contract audit。 |
| `grvt` | GRVT 当前公开交易主线为 perpetuals；API asset kinds 包含 PERPETUAL/FUTURE/CALL/PUT，公开站点显示 spot/options 仍需单独核验上线状态。 | `option,perpetual`；REST/WS 为 spec/parser-only。 | 写 `交易所不支持现货`；期权保持当前 adapter 声明但需单独补 greeks/option order 语义。 |
| `hibachi` | Hibachi 官方交易文档是 perpetual contract/orderbook、funding 和 mark/spot index 价格；未见现货订单生命周期。 | `perpetual`；公共 market/orderbook native，写接口 spec-only。 | 写 `交易所不支持现货`；私有读写和 WS runtime 继续保留 Unsupported。 |
| `lighter` | Lighter 官方 API 以 perpetual futures 为主，同时 WebSocket/API 文档出现 spot market stats 和 spot market 结构。 | `perpetual`；REST/WS spec/parser-only。 | 不能写交易所不支持现货；写 `项目未实现 Spot` 或“当前 `lighter` adapter 未接 spot”。 |
| `mango_markets` | Mango v4 是 Solana on-chain protocol；审计资料说明有 perpetual futures 和 Spot through OpenBook。 | `margin,perpetual`；G0 scan-only。 | 不能写交易所不支持现货；写 `项目未实现 Spot/OpenBook` 或“当前 adapter 不接入 spot”。 |
| `modetrade` | Mode Trade 是 Mode Perpetuals DEX / Orderly EVM profile。未见 ModeTrade-native spot exchange API。 | `perpetual`；只解析 Orderly public info。 | 写 `交易所不支持现货`；私有读写和 WS runtime 继续等 account audit。 |

## 转入任务

| 类型 | adapter | 写法 |
| --- | --- | --- |
| 明确交易所不支持 | `cod3x`, `derive_chain_perps`, `equation`, `zeta_markets`, `aark`, `aftermath`, `d8x`, `grvt`, `hibachi`, `modetrade` | 单交易所文档产品线表写 `交易所不支持现货`。 |
| 官方支持但项目未实现 | `bsx`, `lighter`, `mango_markets` | 写 `项目未实现 Spot`，后续决定是否新增 spot profile/adapter。 |
| 停运边界 | `zeta_markets` | 不补 live WS/trading；公共数据只保留 legacy scan 和 fixture。 |
