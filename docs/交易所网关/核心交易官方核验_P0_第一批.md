# 核心交易官方核验 P0 第一批

状态日期：2026-06-08

本文件核验下单、撤单、查单、订单状态、client order id、TIF 和 market/limit 支持。结论只处理核心交易接口；公共 WebSocket、产品线、仓位、账户和费率仍以各自核验文件为准。

## 核验结论

| adapter | 官方核心交易接口 | 订单类型 / TIF | client order id | 当前项目状态 | 结论 |
| --- | --- | --- | --- | --- | --- |
| `apex` | ApeX Omni API 官方 Trade 文档支持 `POST /v3/order` 创建订单、`POST /v3/delete-order` 撤单、`GET /v3/order` 查单、`GET /v3/open-orders`。 | 支持 Market/Limit/Conditional；TIF 包含 `GOOD_TIL_CANCEL`、`FILL_OR_KILL`、`IMMEDIATE_OR_CANCEL`；Market order 也必须提供 price/worst-price，因为需要 zkLink 签名。 | 支持 `clientOrderId`。 | `place_order=不支持`，`cancel_order=离线`；adapter 文档已写 `POST /api/v3/order` 需要 Rust zkLink/L2 signer。 | 官方支持，项目未实现完整下单 runtime；不能写交易所不支持下单/撤单。补 deterministic zkLink/L2 signer、下单签名向量、撤单 runtime 和 WS/REST 订单确认。 |
| `bybiteu` | Bybit EU UTA 帮助文档把 order creation 映射到 `/v5/order/create`，order deletion 映射到 `/v5/order/cancel`；Bybit V5 官方文档支持 Spot/Margin/USDT Perp/USDC Perp/Futures/Options 创建订单、撤单、全撤。 | 支持 Limit/Market；TIF 包含 `GTC`、`IOC`、`FOK`、`PostOnly`、`RPI`；支持 TP/SL、reduce-only、conditional order。 | 支持 `orderLinkId`，最长 36 字符；同时传 `orderId` 和 `orderLinkId` 时优先 `orderId`。 | 当前 Bybit EU profile 明确禁用 private REST/WS 和真实下单/撤单。 | 官方/区域文档有交易接口，但当前项目因区域凭证和合规 scope 未启用；写 `项目未实现核心交易接口`，不是 `交易所不支持`。 |
| `cex` | CEX.IO Spot Trading REST/WS 支持 `do_new_order`、`do_cancel_my_order`、`do_cancel_all_orders`；旧 REST 也有 `place_order`/`cancel_order`。 | Spot Trading 支持 Market、Limit、Stop Limit；TIF 包含 GTC、IOC、GTD；Market 只允许 IOC。 | 支持 `clientOrderId` 和 `cancelRequestId`。 | 当前 adapter 只实现 public REST；private writes 是 request-spec/unsupported boundary。 | 官方支持 Spot 下单/撤单，项目未实现 private trading runtime；不能写交易所不支持写接口。 |

## 当前项目识别修正

本批同时修正了自动矩阵生成器的证据提取逻辑：

- `bingx.place_order` 这类带 adapter 前缀的 operation 现在会归一化成 `place_order`。
- `spot_place_order`、`perp_cancel_order` 等 product 前缀 operation 现在会归一化成标准能力。
- `operations:` 嵌套结构会展开成 endpoint evidence，AscendEX 这类 mapping 不再被误判为缺下单/撤单。

修正后，`ascendex`、`bingx`、`phemex` 的核心交易接口从剩余队列中移除，因为当前项目已有运行级或映射级证据。它们仍然可以继续做 live-readonly/live-dry-run 验证，但不应再作为“官方是否支持下单/撤单”的待核验项。

## 下一步实现建议

1. `bybiteu` 先补 private REST request specs 和区域凭证 scope guard；未通过区域凭证审计前继续禁用 runtime。
2. `cex` 先补 Spot Trading private REST/WS auth、`do_new_order`/`do_cancel_my_order` fixture 和只读/小额 dry-run gate。
3. `apex` 的关键前置是 zkLink/L2 deterministic signer；没有 signer 不应启用真实 `place_order`。
4. 后续继续从剩余 P2 队列挑核心交易或仓位接口，不再重复核验 `ascendex`、`bingx`、`phemex` 的下单/撤单支持性。
