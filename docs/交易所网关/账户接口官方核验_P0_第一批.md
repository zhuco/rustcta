# 账户接口官方核验 P0 第一批

状态日期：2026-06-08

本批从 `account_state_official_check` 队列先收口 15 个 adapter。判断口径：

- 官方私有 API、SDK 或当前 adapter 文档已经记录余额/账户 readback，但项目 runtime 仍是 request-spec-only、offline 或 unsupported，写 `官方支持，项目未实现/未启用`。
- 当前 adapter/profile 是路由壳、链上审计 profile、已停运 legacy scan 或不承接本地账户模型，写 `交易所不支持当前账户接口 runtime`。

## 核验结论

| adapter | 产品线 | 官方/项目结论 | 项目当前状态 | 写法 |
| --- | --- | --- | --- | --- |
| `bitteam` | Spot | BIT.TEAM OpenAPI/Swagger 列出 `GET /trade/api/ccxt/balance`。 | 余额为 request-spec-only，runtime 返回 unsupported。 | 官方支持，项目未实现/未启用。 |
| `p2b` | Spot | P2B REST 文档列出 `POST /api/v2/account/balances`。 | 余额为 request-spec-only，runtime 返回 unsupported。 | 官方支持，项目未实现/未启用。 |
| `yobit` | Spot | YoBit TAPI `getInfo` 可作为资金/余额来源。 | 余额为 request-spec-only，runtime 返回 unsupported。 | 官方支持，项目未实现/未启用。 |
| `zebpay` | Spot/Futures profile | ZebPay private docs 列出 `GET /wallet/balance?trade_pair=...`。 | bearer/onboarding/token lifecycle 未启用，private runtime unsupported。 | 官方支持，项目未实现/未启用。 |
| `latoken` | Spot/Futures profile | LATOKEN v2 private API 有 `/v2/auth/account` 账户接口。 | private REST live read/write disabled，仅保留 request-spec/signing fixture。 | 官方支持，项目未实现/未启用。 |
| `hollaex` | Spot | HollaEx official API profile 有 `GET /user/balance`。 | offline request-spec only，runtime private read remains unsupported。 | 官方支持，项目未实现/未启用。 |
| `bitbns` | Spot/Margin | 官方 SDK/site 有 private API、API Trading 和账户 readback 线索。 | scan-only public REST；private account/readback disabled。 | 官方支持，项目未实现/未启用。 |
| `cex` | Spot | CEX.IO private REST/Spot Trading API 有 private balances/orders/fills 口径。 | private balances/orders/fills unsupported，需 read-only request-spec promotion。 | 官方支持，项目未实现/未启用。 |
| `hitbtc` | Spot/Margin/Futures profile | HitBTC v3 documents wallet/private endpoints；wallet WS 也单独存在。 | private readbacks 和 wallet runtime 均 disabled。 | 官方支持，项目未实现/未启用。 |
| `fmfwio` | Spot/Margin/Futures profile | FMFW.io HitBTC-family API documents wallet/private endpoints。 | private readbacks 和 wallet runtime 均 disabled。 | 官方支持，项目未实现/未启用。 |

| `cod3x` | Perpetual routing | Cod3x-native profile 未建立稳定本地账户/余额 API；下游 venue account 走各自 adapter。 | `get_balances=不支持`。 | 交易所不支持当前账户接口 runtime。 |
| `derive_chain_perps` | Perpetual profile | Derive Exchange account runtime 归 `derive` adapter；本 profile 只记录 Derive Chain settlement boundary。 | `get_balances=不支持`。 | 交易所不支持当前账户接口 runtime。 |
| `equation` | Perpetual | Equation 当前 adapter 是 audit-only EVM protocol profile，账户/抵押状态源未提升为 exchange-gateway runtime。 | `get_balances=不支持`。 | 交易所不支持当前账户接口 runtime。 |
| `zeta_markets` | Perpetual legacy | 原 Zeta venue 已停止运营；legacy adapter 只保留 public scan。 | `get_balances=不支持`。 | 交易所不支持当前账户接口 runtime。 |

## 后续实现边界

- `bitteam`、`p2b`、`yobit`、`zebpay`、`latoken`、`hollaex`：先补 read-only auth smoke、余额 request spec/parser，再允许 private read runtime。
- `bitbns`、`cex`、`hitbtc`、`fmfwio`、`paymium`：先把官方账户/余额 endpoint 明确进 endpoint mapping，再补签名、parser、credential scope 和只读对账。
- `cod3x`、`derive_chain_perps`、`equation`、`zeta_markets`：保持当前 profile-local unsupported；不要把下游 venue、链上示例或 legacy SDK 账户状态提升为 live gateway runtime。
