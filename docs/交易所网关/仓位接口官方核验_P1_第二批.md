# 仓位接口官方核验 P1 第二批

状态日期：2026-06-08

本批收口剩余 13 个 `position_official_check`。判断口径：

- 当前项目已经有 `get_positions` runtime、request/parser 测试或 endpoint mapping 证据的，不新增补全任务，只修正矩阵识别和单交易所文档。
- 官方/profile 有仓位能力但当前 `huobi` legacy profile 没有 `get_positions` 运行证据的，写 `官方支持，项目未实现/未启用`。
- 当前 profile 不承接稳定 live positions runtime 的，写 `交易所不支持当前仓位接口 runtime`。

## 后续实现边界

- `huobi`：不要直接在矩阵里把 `htx` runtime 算作 `huobi` 已实现；需要补 Huobi legacy profile 的 `get_positions` endpoint mapping、request spec、parser/delegation 证据和 `positions_cross` REST/WS 对账说明。
- `derive_chain_perps`、`equation`：保持 profile-local unsupported；不要复制 `derive` 或链上协议示例到 live gateway runtime。
- `oxfun`：保持 positions unsupported，直到官方 private positions endpoint、签名权限、分页/字段和 live auth 全部验证。
- `zeta_markets`：保持 legacy scan-only，不补 live private position runtime。
