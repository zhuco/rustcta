# 账户接口官方核验 P1 第二批

状态日期：2026-06-08

本批收口 `account_state_official_check` 剩余 17 个 adapter。判断口径延续第一批：

- 官方私有 API、SDK、链上账户模型或当前 adapter 文档已经记录余额/抵押品/账户权益 readback，但项目 runtime 仍是 disabled、request-spec-only、offline 或 unsupported，写 `官方支持，项目未实现/未启用`。
- 当前官方资料未核到稳定余额/账户权益 endpoint，或当前 profile 不承接本地账户 runtime，写 `交易所不支持当前账户接口 runtime`。

## 后续实现边界

- `aark`、`modetrade`：复用 Orderly signed account read path，但必须先完成各 profile 的 account id、Ed25519 signing、credential scope 和 read-only auth smoke。
- `aftermath`、`d8x`、`mango_markets`、`wavesexchange`：先补链上账户/钱包读取模型、indexer 或 node fallback，再把 collateral/equity/balances 映射成 gateway account state。
- `bybiteu`、`myokx`、`okxus`：按区域 profile 单独做 domain、API key scope、KYC/eligibility 和 read-only guard；不能直接打开全球站 runtime。
- `bsx`、`grvt`、`hibachi`、`lighter`、`paradex`：先补账户认证/session/JWT/request spec/parser，再接 REST 或 private WS 对账。
- `bitkan`、`oxfun`、`pacifica`：保持当前账户/余额 unsupported；只有找到稳定官方 balances/account-equity endpoint 后才重新核验。
