## P0 公共订单簿 WebSocket

这些直接影响跨所套利上线。每个任务至少要补：

- `endpoint_mapping.yaml` 结构化字段：channel、订阅方式、推流间隔、档位、sequence/checksum、snapshot 重建。
- WebSocket 订阅 payload 和 parser fixture。
- REST snapshot + delta 重建策略。
- 断线重连后的重新订阅和 book resync。

| adapter | 产品线 | 当前项目状态 | 必补内容 |
| --- | --- | --- | --- |
| `binance` | Spot | public WS unsupported | Spot depth/bookTicker、100ms/1000ms、5/10/20 档、update id 重建。 |
| `okx` | Spot/Swap/Futures/Option | public WS unsupported | books5、bbo-tbt、10ms/100ms、VIP 限制 channel 标注。 |
| `bitget` | Spot | public WS unsupported | books1/books5/books15、10ms/200ms、CRC32。 |
| `gateio` | Spot | public WS unsupported | `spot.order_book_update`、20ms/100ms、20/100 档。 |
| `mexc` | Spot | public WS unsupported | 10ms/100ms depth、bookTicker、fromVersion/toVersion。 |
| `binanceus` | Spot | public WS unsupported | 按 Binance Spot 同类补 100ms depth/bookTicker。 |
| `binancecoinm` | COIN-M | public WS unsupported | COIN-M partial/diff depth、100/250/500ms、`U/u/pu`。 |
| `bybit` | Spot/Linear/Inverse/Option | spec_only | 1/50/200/1000 档 interval、`seq`、snapshot/delta。 |
| `kucoin` | Spot/Futures | native 但缺细项 | `obu` BBO/5/50/increment、sequence、snapshot replay；2026-06-17 UTC 后补 UTA `depth=increment@10ms` 500 档迁移。 |
| `htx` / `huobi` | Spot/USDT-M | declared/parser evidence | Spot MBP、合约 high_freq depth、seqNum/prevSeqNum/version。 |
| `lbank` | Spot | native 但缺细项 | 10/50/100 档、官方未给固定 ms、无 sequence 风险。 |
| `coinw` | Spot/Futures | parser_only | Spot snapshot/incremental depth；Futures 100 档；seq/startSeq/endSeq。 |

## 验收标准

每个补全任务完成时，至少满足：

1. 单交易所文档写清官方产品线结论，未支持项使用 `交易所不支持`，项目缺口使用 `项目未实现`。
2. `endpoint_mapping.yaml` 包含对应 REST/WS operation、support、product、market_types、auth、rate_limit_bucket。
3. WebSocket 行情包含订阅方式、推流间隔、档位、sequence/checksum、snapshot 重建。
4. 至少补一个成功 parser fixture；写接口还要补 request spec 和 signing vector。
5. 重新运行 `python3 scripts/generate_exchange_gateway_matrix.py`，确认矩阵状态变化符合预期。
6. 定向运行对应 adapter 测试；公共逻辑变化再跑 `cargo check -p rustcta-exchange-gateway --lib`。
